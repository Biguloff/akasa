Telegram-бот = точка входа — обрабатывает команды, отправляет уведомления, запускает WebApp. WebApp — полноценный React SPA внутри Telegram для сложного UI (дашборды, графики, календарь). Оба клиента используют единый API с JWT-авторизацией.

🔄 Потоки данных

  • Фото тела: Клиент → API → fal.ai (vision) → анализ → БД
  • Фото еды: Клиент → API → fal.ai + FatSecret → БЖУ → БД
  • Рекомендации: БД (профиль) → OpenRouter (LLM) → план → клиент
  • Пуши: Scheduler (Redis) → Bot API → Telegram пользователю
  • Продукция: API каталога Akasa → контекстный матчинг → рекомендация

05 — Единый аккаунт

SSO: один логин —
все платформы

Telegram initData как основная авторизация. JWT токены для всех клиентов. Привязка email для веб-доступа.

Telegram Mini App               Веб-сайт Akasa              Mobile App
       │                              │                          │
       │ initData                     │ email + password         │ OAuth / biometric
       │ (подпись бота)                │ (или OAuth)               │
       ▼                              ▼                          ▼
┌──────────────────────────────────────────────────────────────────────┐
│                     Auth Service (FastAPI)                          │
│                                                                      │
│   1. Валидация initData / email+pass / OAuth                         │
│   2. Найти или создать пользователя                                 │
│   3. Выдать JWT access_token (единый формат для всех клиентов)         │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
       │
       ▼
┌──────────────────┐
│  users таблица    │    Один пользователь = telegram_id + email + phone
│  auth_tokens      │    Множество провайдеров привязаны к одному user_id
└──────────────────┘
Привязка аккаунтов: Пользователь приходит через Telegram → автоматически создаётся аккаунт. Потом может привязать email для входа на сайт. Или наоборот — зарегистрировался на сайте, потом привязал Telegram. Один профиль, полная история.
06 — AI-стек

AI и внешние
API

Комбинация сервисов для анализа фото, распознавания еды и генерации рекомендаций.

Сервис Задача Стоимость Альтернатива
fal.ai Анализ фото тела, распознавание еды (computer vision) ~$0.001-0.01 / запрос Clarifai, Foodvisor
OpenRouter LLM — генерация планов питания, тренировок, мотивация ~$0.001-0.03 / запрос (зависит от модели) OpenAI API, Claude API
FatSecret API База продуктов, калории, БЖУ (текстовый поиск) Бесплатно (до 10K запросов/день) Nutritionix ($299/мес)
Telegram Bot API Бот, уведомления, WebApp, платежи Бесплатно —
ЕЖЕМЕСЯЧНЫЕ РАСХОДЫ НА AI API
15 — 40K ₽
в месяц при 500-2000 активных пользователей
Масштабируется линейно. При 10K пользователей — ~80-150K ₽/мес. FatSecret бесплатен до 10K запросов/день.
07 — Роадмап

Дорожная карта
разработки

5 фаз: от Telegram-бота до полноценной клубной платформы

Фаза 1 — Месяцы 1-3

Telegram-бот + Базовый AI

  • Регистрация + SSO (Telegram)
  • Онбординг (цели, параметры)
  • Загрузка фото тела
  • AI-анализ (fal.ai)
  • Генерация плана питания (OpenRouter)
  • Генерация плана тренировок
  • Распознавание еды по фото
  • Каталог продукции Akasa
  • Проактивные сообщения
  • Базовый трекинг веса
Результат: Рабочий Telegram-бот, готовый к запуску с первыми пользователями (бета)
Фаза 2 — Месяцы 3-5

Telegram WebApp + Клуб

  • Telegram Mini App (React)
  • Дашборд прогресса
  • Графики (вес, замеры, БЖУ)
  • Фото до/после
  • Членство в Akasa Club (уровни)
  • Telegram Payments (подписки)
  • Баллы за активность
  • Реферальная система
Результат: Полноценный Mini App внутри Telegram + клубная система с уровнями
Фаза 3 — Месяцы 5-7

Пуши, мероприятия, активности

  • Мероприятия и челленджи
  • Утренние зарядки (контент)
  • Розыгрыши для членов клуба
  • Эксклюзивные скидки и промокоды
  • Расписание мероприятий в WebApp
  • Уведомления о новых продуктах
  • Геймификация (достижения, бейджи)
Результат: Живое комьюнити с активностями, челленджами и вовлечением
Фаза 4 — Месяцы 7-9

Веб-версия + Админка

  • Веб-сайт (Next.js) с полным функционалом
  • Привязка email к аккаунту
  • Админ-панель для команды Akasa
  • Аналитика конверсий (рекомендации → покупки)
  • Управление мероприятиями
  • CRM-данные о пользователях
  • Управление каталогом продукции
Результат: Полноценная веб-платформа + инструменты управления для Akasa
Фаза 5 — Месяцы 10-13

Нативное приложение

  • iOS + Android (React Native)
  • Push-уведомления
  • Apple Health / Google Fit
  • Офлайн-режим
  • Камера для фото еды (нативная)
  • Виджеты на главный экран
  • Биометрическая авторизация
Результат: Полная экосистема: Telegram + Web + Mobile. Единый аккаунт. Akasa Club.
