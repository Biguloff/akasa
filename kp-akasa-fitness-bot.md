# Коммерческое предложение
## AI Фитнес-Ассистент для экосистемы Akasa

---

## 1. ATTENTION — О проекте

> Спортивное питание без сопровождения — это как абонемент в зал без тренера. Работает для 10% мотивированных. Остальные 90% уходят.

**Мы предлагаем:** создать AI-ассистента, который превращает каждого покупателя Akasa в постоянного клиента, давая ему то, чего нет ни у одного конкурента — **персонального AI-тренера и нутрициолога**, встроенного в экосистему бренда.

**Формат:** Telegram-бот (MVP) → Веб-приложение → Нативное мобильное приложение (iOS/Android)

---

## 2. INTEREST — Проблема рынка

### Почему клиенты уходят

| Проблема | Последствие для Akasa |
|----------|----------------------|
| Купил спортпит, но не знает как правильно принимать | Не видит результата → "продукт не работает" |
| Начал тренироваться без плана | Травмы, отсутствие прогресса → бросает |
| Нет мотивации и сопровождения | Забывает про режим → перестаёт покупать |
| Не понимает своё питание | Ест неправильно даже с добавками → нет эффекта |

### Что делают конкуренты

Большинство производителей спортпитания ограничиваются:
- Статьями в блоге (никто не читает)
- Контентом в соцсетях (теряется в ленте)
- Общими рекомендациями на упаковке

**Никто** не предлагает персонализированное AI-сопровождение, привязанное к своей продукции.

## 3. DESIRE — Решение

### 3.1 Как это работает для пользователя

```
Шаг 1: РЕГИСТРАЦИЯ
├── Единый аккаунт Akasa (SSO)
├── Заходит в Telegram-бот
└── Указывает цели (похудение / набор массы / поддержание формы)

Шаг 2: АНАЛИЗ
├── Загружает фото тела
├── Вводит параметры (вес, рост, возраст, активность)
└── AI анализирует текущее состояние

Шаг 3: ПЕРСОНАЛЬНЫЙ ПЛАН
├── Генерация плана питания (с учётом продукции Akasa)
├── Генерация плана тренировок
└── Расписание приёма добавок

Шаг 4: ЕЖЕДНЕВНОЕ СОПРОВОЖДЕНИЕ
├── Бот сам пишет: "Как прошла тренировка?"
├── Пользователь фоткает еду → AI считает калории/БЖУ
├── Напоминания о приёме пищи, тренировке, добавках
└── Корректировка плана на основе прогресса

Шаг 5: ТРЕКИНГ ПРОГРЕССА
├── Графики веса и замеров
├── Фото до/после
├── Еженедельные отчёты
└── Рекомендации по корректировке
```

### 3.2 Функциональные блоки

#### Блок 1: AI-анализ тела
- Загрузка фото и параметров пользователя
- AI-оценка текущей формы и состава тела
- Определение реалистичных целей и сроков
- **Технология:** fal.ai (computer vision)

#### Блок 2: Распознавание еды по фото
- Пользователь фотографирует тарелку с едой
- AI определяет блюда и ингредиенты
- Автоматический расчёт калорий, белков, жиров, углеводов
- Сравнение с дневным планом питания
- **Технология:** fal.ai (image recognition) + LLM (OpenRouter)

#### Блок 3: Персональные планы тренировок
- Генерация программы тренировок под цели пользователя
- Учёт уровня подготовки и доступного оборудования
- Адаптация нагрузки на основе обратной связи
- Видео/описания упражнений
- **Технология:** LLM (OpenRouter)

#### Блок 4: Трекинг прогресса
- Ежедневный учёт питания и тренировок
- Графики: вес, обхваты, процент жира
- Фото-дневник (до/после, промежуточные)
- Еженедельные и месячные отчёты с аналитикой
- **Технология:** PostgreSQL + аналитический движок

#### Блок 5: Каталог продукции Akasa
- Интеграция с каталогом спортивного питания Akasa
- Контекстные рекомендации продукции в рамках плана питания
- Персональный подбор добавок под цели пользователя
- Ссылки на покупку (deeplink на сайт/магазин Akasa)
- **Технология:** API интеграция с каталогом Akasa

#### Блок 6: Реферальная система
- Уникальные реферальные ссылки для каждого пользователя
- Бонусы за приглашённых друзей (скидки, бесплатный период)
- Статистика приглашений
- Геймификация (уровни, достижения)

### 3.3 Единая учётная запись (SSO)

Пользователь создаёт **один аккаунт Akasa**, который работает:
- На сайте Akasa
- В Telegram-боте
- В веб-приложении
- В будущем мобильном приложении
- В любых других сервисах экосистемы

Это обеспечивает:
- Бесшовный опыт для пользователя
- Единый профиль клиента для Akasa
- Полную картину взаимодействий с брендом

### 3.4 Проактивное взаимодействие

Ключевое отличие от обычных фитнес-ботов — **бот сам инициирует общение**:

- Утром: "Доброе утро! Вот твой план питания на сегодня"
- Перед тренировкой: "Через час тренировка ног. Не забудь разминку!"
- После тренировки: "Как прошла тренировка? Оцени от 1 до 10"
- Вечером: "Ты сегодня съел 1800 из 2200 ккал. Можешь добавить перекус"
- При отсутствии: "Тебя не было 3 дня. Всё в порядке? Давай скорректируем план"

## 4. Техническая архитектура

### Стек технологий

| Компонент | Технология | Обоснование |
|-----------|-----------|-------------|
| AI (изображения) | fal.ai | Быстрый inference, анализ фото тела и еды |
| AI (текст/рекомендации) | OpenRouter (мультимодельный) | Гибкость, выбор лучшей модели под задачу |
| Backend | Python (FastAPI) | Высокая производительность, async, AI-экосистема |
| База данных | PostgreSQL | Надёжность, аналитика, JSON-поддержка |
| Кэш/очереди | Redis | Сессии, очереди задач, кэширование |
| Telegram | Telegram Bot API (aiogram) | Стабильная библиотека, async |
| Веб-фронтенд | React / Next.js | SSR, производительность, экосистема |
| Авторизация (SSO) | OAuth 2.0 / OpenID Connect | Индустриальный стандарт, безопасность |
| Хранение файлов | S3-совместимое хранилище | Фото пользователей, медиа |
| Инфраструктура | Docker + CI/CD | Масштабируемость, автоматизация деплоя |

### Схема взаимодействия

```
┌─────────────┐     ┌─────────────┐     ┌──────────────┐
│  Telegram    │     │  Web App    │     │  Mobile App  │
│  Bot         │     │  (React)   │     │  (будущее)   │
└──────┬───────┘     └──────┬──────┘     └──────┬───────┘
       │                    │                    │
       └────────────┬───────┘────────────────────┘
                    │
              ┌─────▼──────┐
              │   API       │
              │  (FastAPI)  │
              └─────┬───────┘
                    │
       ┌────────────┼────────────┐
       │            │            │
┌──────▼───┐ ┌─────▼─────┐ ┌───▼────────┐
│ fal.ai   │ │ OpenRouter │ │ PostgreSQL │
│ (vision) │ │ (LLM)     │ │ + Redis    │
└──────────┘ └───────────┘ └────────────┘
```

### База данных — ключевые сущности

- **Users** — профиль, параметры, цели, SSO-привязка
- **Body Metrics** — вес, замеры, фото (история)
- **Meal Logs** — дневник питания (фото, распознанные продукты, БЖУ)
- **Training Plans** — персональные программы тренировок
- **Training Logs** — выполненные тренировки, обратная связь
- **Product Recommendations** — рекомендации продукции Akasa, конверсии
- **Referrals** — реферальные связи, бонусы
- **Notifications** — расписание и история уведомлений

## 5. DESIRE (усиление) — Бизнес-выгоды для Akasa

### Прямые выгоды

| Выгода | Описание |
|--------|----------|
| **Рост повторных продаж** | Бот рекомендует продукцию Akasa в контексте персонального плана — не как рекламу, а как часть программы |
| **Увеличение LTV** | Пользователь остаётся в экосистеме месяцами, а не уходит после первой покупки |
| **Данные о клиентах** | Цели, диета, активность, предпочтения — точный портрет для маркетинга и разработки новых продуктов |
| **Виральный рост** | Реферальная система + "результаты до/после" = органическое привлечение |
| **Конкурентный барьер** | Первый производитель спортпита с персональным AI-ассистентом в СНГ |

### Аналитика и инсайты

Вся активность пользователей сохраняется в базе данных, что даёт Akasa:
- Понимание, какие продукты чаще всего покупают в связке
- Данные о том, какие цели ставят клиенты (похудение vs набор массы)
- Паттерны питания целевой аудитории
- Эффективность рекомендаций (конверсия в покупку)
- Сегментация клиентов для таргетированных кампаний

## 6. ACTION — Этапы разработки

### Фаза 1: Telegram MVP

**Цель:** Запустить рабочего бота с базовым AI-функционалом

- Telegram-бот с регистрацией и SSO
- Загрузка фото и параметров тела
- AI-анализ и генерация планов питания/тренировок
- Распознавание еды по фото
- Интеграция каталога продукции Akasa
- Базовый трекинг прогресса
- Проактивные сообщения и напоминания

### Фаза 2: Веб-приложение

**Цель:** Расширить аудиторию за пределы Telegram

- Веб-версия с полным функционалом бота
- Расширенная аналитика и дашборды прогресса
- Детализированные графики и отчёты
- Реферальная система с личным кабинетом
- Админ-панель для команды Akasa

### Фаза 3: Нативное приложение

**Цель:** Полноценная мобильная платформа

- iOS и Android приложение
- Push-уведомления
- Офлайн-режим (базовый)
- Интеграция с фитнес-трекерами (Apple Health, Google Fit)
- Расширенная геймификация

## 7. ACTION — Условия и следующие шаги

### Формат работы
- Итеративная разработка (Agile/Scrum)
- Регулярные демо и отчёты о прогрессе
- Совместное определение приоритетов

### Стоимость
Обсуждается отдельно после согласования объёма работ и приоритетов по функционалу для каждой фазы.

### Следующие шаги

1. **Встреча** — обсуждение приоритетов и деталей (30-60 минут)
2. **Детализация ТЗ** — на основе согласованных приоритетов
3. **Согласование бюджета и сроков** — по каждой фазе отдельно
4. **Старт разработки** — Фаза 1 (Telegram MVP)
